# 数据库设计文档

## 一、数据库基本信息

**数据库名：** galgame_db
## 二、字段详细说明

### 1. user 表（用户，预留）

| 字段名    | 数据类型    | 说明                               |
| :-------- | :---------- | :--------------------------------- |
| id        | BIGINT      | 用户ID，自增主键                   |
| username  | VARCHAR(50) | 用户名，唯一且非空                 |
| password  | VARCHAR(100)| 加密后的密码（如 BCrypt 加密），非空 |
| created_at| DATETIME    | 注册时间，非空                     |

**说明：** 当前暂不实现用户认证，但预留此表以便未来扩展。游客存档时，save 表中的 user_id 为 NULL，使用 visitor_id 标识。

### 2. game 表（游戏基本信息）

| 字段名    | 数据类型    | 说明                               |
| :-------- | :---------- | :--------------------------------- |
| id        | BIGINT      | 游戏ID，自增主键                   |
| title     | VARCHAR(100)| 游戏标题，非空                     |
| cover_url | VARCHAR(255)| 封面图片的 URL（可为空）           |
| description| TEXT       | 游戏简介（可为空）                 |
| tags      | VARCHAR(255)| 标签，多个用逗号分隔的字符串（简化方案，可为空） |
| status    | TINYINT     | 游戏状态：0-下架，1-上架，默认 1，非空 |
| created_at| DATETIME    | 创建时间，非空                     |

**索引：** (status, created_at) 联合索引加速按状态过滤并按时间排序的查询。

### 3. scene 表（游戏场景/剧本）

| 字段名    | 数据类型    | 说明                               |
| :-------- | :---------- | :--------------------------------- |
| id        | BIGINT      | 场景ID，自增主键                   |
| game_id   | BIGINT      | 所属游戏ID，外键关联 game(id)，非空，删除游戏时级联删除场景 |
| scene_key | VARCHAR(50) | 场景唯一标识（如 "start", "scene_1"），非空 |
| content   | JSON        | 剧本内容（JSON 格式），非空        |
| next_scene| VARCHAR(50) | 默认下一场景键（无选项时使用），可为空 |

**约束：**
- 外键：game_id 引用 game(id)，ON DELETE CASCADE
- 唯一约束：(game_id, scene_key) 确保每个游戏内的场景键唯一

**content JSON 示例：**

```json
{
  "bg": "bg_school.jpg",
  "characters": [
    { "name": "alice", "image": "alice_happy.png", "position": "left" }
  ],
  "text": "今天天气真好。",
  "next": "scene2",
  "options": [
    { "text": "去公园", "next": "scene_park" },
    { "text": "去图书馆", "next": "scene_library" }
  ]
}
```

**说明：** 无选项时，使用 next 字段；有选项时，使用 options 数组，next 可忽略。

### 4. save 表（存档）

| 字段名    | 数据类型    | 说明                               |
| :-------- | :---------- | :--------------------------------- |
| id        | BIGINT      | 存档ID，自增主键                   |
| user_id   | BIGINT      | 注册用户ID（关联 user.id），可为 NULL |
| visitor_id| VARCHAR(36) | 游客ID（UUID 格式），可为 NULL     |
| game_id   | BIGINT      | 游戏ID，外键关联 game(id)，非空，删除游戏时级联删除存档 |
| scene_key | VARCHAR(50) | 当前场景键，非空                   |
| variables | JSON        | 存档变量（如好感度、道具等），可为 NULL |
| updated_at| DATETIME    | 最后更新时间，非空                 |

**约束：**
- 外键：game_id 引用 game(id)，ON DELETE CASCADE
- 外键：user_id 引用 user(id)，ON DELETE SET NULL（用户删除后，存档保留但 user_id 置为 NULL）
- 业务规则：user_id 和 visitor_id 不能同时为 NULL（需在应用层保证）

**索引：**
- (game_id, user_id) 联合索引，加速查询某游戏下注册用户的存档
- (game_id, visitor_id) 联合索引，加速查询某游戏下游客的存档
- updated_at 索引，便于按时间排序

**variables 示例：**

```json
{ "好感度": 80, "道具": ["钥匙", "照片"], "flag_01": true }
```

## 三、建表 SQL 语句

```sql
-- 创建数据库（如果不存在）
CREATE DATABASE IF NOT EXISTS galgame_db
CHARACTER SET utf8mb4
COLLATE utf8mb4_unicode_ci;

USE galgame_db;

-- 1. 用户表（预留）
CREATE TABLE user (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    username VARCHAR(50) NOT NULL UNIQUE,
    password VARCHAR(100) NOT NULL,
    created_at DATETIME NOT NULL
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 2. 游戏表
CREATE TABLE game (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    title VARCHAR(100) NOT NULL,
    cover_url VARCHAR(255),
    description TEXT,
    tags VARCHAR(255),
    status TINYINT NOT NULL DEFAULT 1,
    created_at DATETIME NOT NULL,
    INDEX idx_status_created (status, created_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 3. 场景表
CREATE TABLE scene (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    game_id BIGINT NOT NULL,
    scene_key VARCHAR(50) NOT NULL,
    content JSON NOT NULL,
    next_scene VARCHAR(50),
    FOREIGN KEY (game_id) REFERENCES game(id) ON DELETE CASCADE,
    UNIQUE KEY uk_game_scene (game_id, scene_key)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 4. 存档表（同时支持注册用户和游客）
CREATE TABLE save (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT,
    visitor_id VARCHAR(36),
    game_id BIGINT NOT NULL,
    scene_key VARCHAR(50) NOT NULL,
    variables JSON,
    updated_at DATETIME NOT NULL,
    FOREIGN KEY (game_id) REFERENCES game(id) ON DELETE CASCADE,
    FOREIGN KEY (user_id) REFERENCES user(id) ON DELETE SET NULL,
    INDEX idx_game_user (game_id, user_id),
    INDEX idx_game_visitor (game_id, visitor_id),
    INDEX idx_updated (updated_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```
