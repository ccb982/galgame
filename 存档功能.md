## 一、存档的基本概念
存档 是游戏状态的快照，用于保存玩家的游戏进度，以便后续继续游戏。在 Galgame 播放器中，存档包含以下核心信息：

1. 游戏标识 ：当前游戏的 ID
2. 场景标识 ：当前所在场景的 key
3. 游客标识 ：玩家的唯一标识（用于区分不同玩家的存档）
4. 游戏变量 ：游戏中累积的变量状态（如选择结果、角色关系等）
5. 时间戳 ：存档创建/更新的时间
## 二、存档的数据结构
### 1. 后端数据库结构（Save 实体类）
Save.java ：

```
@Entity
@Table(name = "save")
@Data
@NoArgsConstructor
@AllArgsConstructor
public class Save {
    @Id
    @GeneratedValue(strategy = 
    GenerationType.IDENTITY)
    private Long id;

    @Column(name = "game_id", nullable = 
    false)
    private Long gameId;

    @Column(name = "scene_key", nullable = 
    false, length = 50)
    private String sceneKey;

    @Column(name = "visitor_id", nullable = 
    false, length = 100)
    private String visitorId;

    @Column(name = "variables", 
    columnDefinition = "TEXT")
    private String variables; // JSON 格式存储

    @Column(name = "updated_at", nullable = 
    false)
    @Temporal(TemporalType.TIMESTAMP)
    private Date updatedAt;
}
```

## 三、存档的存储与读取流程
### 1. 存档创建流程
前端操作 ：

1. 玩家点击"保存"按钮
2. 前端收集当前游戏状态（游戏 ID、场景 key、游戏变量）
3. 调用 createSave API 创建存档
4. 显示保存成功提示
后端处理 ：

1. 接收存档数据
2. 验证数据完整性
3. 保存到数据库
4. 返回保存结果
### 2. 存档读取流程
前端操作 ：

1. 玩家在游戏详情页选择存档
2. 前端通过 URL 参数传递 saveId 到播放器
3. 播放器加载存档数据
4. 根据存档恢复游戏状态
后端处理 ：

1. 根据 saveId 查询存档
2. 返回存档数据
3. 前端解析数据并恢复游戏状态
## 四、存档的作用机制
### 1. 游戏变量的存储与恢复
游戏变量 是存档的核心价值，用于：

- 记录玩家选择 ：不同选择会影响后续剧情
- 追踪游戏状态 ：如角色关系、物品收集等
- 实现剧情分支 ：根据变量值显示不同剧情
示例 ：

```
// 存档时
const variables = {
  relationship: 80,       // 角色关系值
  hasKey: true,           // 是否获得钥匙
  choices: ["trust", "help"]  // 历史选择
};

// 读取存档后
if (variables.relationship > 70) {
  // 显示友好剧情
} else {
  // 显示敌对剧情
}
```
### 2. 场景定位与恢复
场景 key 用于快速定位到具体场景：

- 确保玩家从上次离开的地方继续
- 支持多个存档点，方便玩家在不同剧情分支间切换
### 3. 游客 ID 的作用
visitorId 确保：

- 不同玩家的存档相互隔离
- 同一玩家在不同设备上的存档一致性（通过 localStorage 持久化）
- 无需登录即可使用存档功能

### 2. 后端 API 实现
SaveController.java ：

```
@RestController
@RequestMapping("/saves")
public class SaveController {
    @Autowired
    private SaveService saveService;

    @GetMapping
    public List<Save> getSaves(@RequestParam 
    Long gameId, @RequestParam String 
    visitorId) {
        return saveService.
        getSavesByGameIdAndVisitorId(gameId, 
        visitorId);
    }

    @PostMapping
    public Save createSave(@RequestBody Save 
    save) {
        return saveService.saveSave(save);
    }

    @PutMapping("/{id}")
    public Save updateSave(@PathVariable 
    Long id, @RequestBody Save save) {
        return saveService.updateSave(id, 
        save);
    }

    @DeleteMapping("/{id}")
    public void deleteSave(@PathVariable 
    Long id) {
        saveService.deleteSave(id);
    }
}
```
## 六、存档的实际应用场景
### 1. 分支剧情管理
示例 ：

- 玩家选择"帮助角色 A" → 关系值增加
- 玩家选择"无视角色 A" → 关系值减少
- 这些选择会存储在存档的变量中
- 后续剧情根据关系值显示不同内容
### 2. 多结局系统
实现方式 ：

- 游戏变量累积到一定值触发不同结局
- 存档保存这些变量状态
- 玩家可以通过不同存档体验不同结局
### 3. 游戏进度管理
优势 ：

- 支持随时保存和加载
- 适合章节较长的 Galgame
- 方便玩家尝试不同选择分支
