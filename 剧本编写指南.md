# 剧本编写指南

## 基本结构

剧本文件为 JSON 格式，包含一个 `scenes` 数组，每个场景都是一个独立的剧情节点。

```json
{
  "scenes": [
    {
      "sceneKey": "场景唯一标识",
      "content": {
        "bg": "背景图片URL",
        "text": "场景文本内容",
        "characters": [],
        "options": []
      }
    }
  ]
}
```

---

## 字段说明

### 1. sceneKey（必需）

场景的唯一标识符，用于选项跳转。

**命名规范：**
- 使用英文小写字母
- 可包含下划线 `_`
- 应具有描述性

**示例：**
```json
"sceneKey": "start"
"sceneKey": "training"
"sceneKey": "coffee_shop"
```

---

### 2. content（必需）

场景内容对象，包含以下字段：

#### bg（背景图片）

**格式：** 字符串，图片URL

**支持类型：**
- 网络图片：`"https://images.unsplash.com/photo-xxx"`
- 本地上传图片：`"/uploads/images/scenes/backgrounds/xxx.webp"`

**示例：**
```json
"bg": "https://images.unsplash.com/photo-1517649763962-0c623066050b?w=1920"
"bg": "/uploads/images/scenes/backgrounds/42e083ec-0389-416e-91d3-1406233bb46b.webp"
```

---

#### text（场景文本）

**格式：** 字符串，显示在对话框中的文本

**支持格式：**
- 普通文本
- 角色对话：`"角色名：'对话内容'"`
- 多行文本：使用 `\n` 换行

**示例：**
```json
"text": "这是一个阳光明媚的下午，篮球场上充满了活力。"
"text": "科比：'今天的训练重点是投篮和防守配合。大家准备好了吗？'"
"text": "旁白：你站在神秘城堡的大门前。\n主角：这座城堡看起来已经废弃很久了。"
```

---

#### characters（角色立绘）

**格式：** 数组，可包含多个角色

**角色对象结构：**
```json
{
  "name": "角色名称",
  "image": "角色图片URL",
  "position": "位置"
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| name | string | 角色名称，显示在角色下方 |
| image | string | 角色立绘图片URL |
| position | string | 位置：`left`、`center`、`right` |

**位置说明：**
- `left`：屏幕左侧
- `center`：屏幕中央
- `right`：屏幕右侧

**示例：**
```json
"characters": [
  {
    "name": "科比",
    "image": "/uploads/images/scenes/characters/bd3ba139-d655-44f3-accf-45116843795d.webp",
    "position": "left"
  },
  {
    "name": "牢e",
    "image": "/uploads/images/scenes/characters/305ef4c0-1ba1-48f2-9267-5ec37076271f.png",
    "position": "center"
  },
  {
    "name": "群友A",
    "image": "/uploads/images/scenes/characters/e15dc78d-be1e-4674-b652-17538071d9aa.png",
    "position": "right"
  }
]
```

**无角色场景：**
```json
"characters": []
```

---

#### options（选项）

**格式：** 数组，玩家可选择的选项

**选项对象结构：**
```json
{
  "text": "选项显示文本",
  "next": "目标场景的sceneKey"
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| text | string | 选项按钮显示的文本 |
| next | string | 点击后跳转的场景标识 |

**示例：**
```json
"options": [
  {
    "text": "开始训练",
    "next": "training"
  },
  {
    "text": "休息一下",
    "next": "rest"
  }
]
```

**单一选项：**
```json
"options": [
  {
    "text": "继续",
    "next": "next_scene"
  }
]
```

**结束场景（重新开始）：**
```json
"options": [
  {
    "text": "重新开始",
    "next": "start"
  }
]
```

## 完整示例

### 简单场景

```json
{
  "sceneKey": "start",
  "content": {
    "bg": "https://images.unsplash.com/photo-15187092688054-9e50e2af4f1a?w=1920",
    "text": "欢迎来到奇幻世界！你是一位勇敢的冒险者。",
    "characters": [],
    "options": [
      {
        "text": "开始冒险",
        "next": "forest"
      }
    ]
  }
}
```

### 多角色场景

```json
{
  "sceneKey": "meeting",
  "content": {
    "bg": "/uploads/images/scenes/backgrounds/castle.webp",
    "text": "骑士：'欢迎来到城堡！'\n公主：'你是来救我的吗？'",
    "characters": [
      {
        "name": "骑士",
        "image": "/uploads/images/scenes/characters/knight.png",
        "position": "left"
      },
      {
        "name": "公主",
        "image": "/uploads/images/scenes/characters/princess.png",
        "position": "right"
      }
    ],
    "options": [
      {
        "text": "是的，我来救你",
        "next": "save_princess"
      },
      {
        "text": "我只是路过",
        "next": "leave"
      }
    ]
  }
}
```

### 分支剧情

```json
{
  "sceneKey": "crossroad",
  "content": {
    "bg": "/uploads/images/scenes/backgrounds/forest_path.webp",
    "text": "你来到了一个岔路口，左边通向森林，右边通向湖泊。",
    "characters": [],
    "options": [
      {
        "text": "向左走（森林）",
        "next": "forest_path"
      },
      {
        "text": "向右走（湖泊）",
        "next": "lake_path"
      },
      {
        "text": "返回",
        "next": "previous_scene"
      }
    ]
  }
}
```

## 编写流程建议

### 1. 规划剧情大纲

先确定：
- 故事起点（start 场景）
- 主要角色
- 分支路线
- 结局数量

### 2. 创建场景流程图

```
start → forest → battle → victory/end
              ↘ escape → cave → treasure/end
```

### 3. 编写场景

按流程顺序编写每个场景，确保：
- `sceneKey` 唯一且有意义
- `next` 指向已存在的 `sceneKey`
- 文本通顺、无错别字

### 4. 检查完整性

- 所有 `next` 指向的场景都存在
- 每个分支都有完整的剧情线
- 至少有一个结局场景

---

## 注意事项

1. **JSON 格式正确**
   - 最后一个元素后不加逗号
   - 字符串使用双引号
   - 使用 JSON 验证工具检查

2. **图片资源**
   - 确保图片 URL 有效
   - 建议使用项目上传的本地图片
   - 图片尺寸建议：背景 1920px 宽，角色透明 PNG

3. **场景跳转**
   - 避免死循环
   - 确保每个分支都能到达结局

4. **文本长度**
   - 避免单行文本过长
   - 使用 `\n` 分段显示长文本

---

## 图片资源路径

上传的图片会保存在以下目录：

| 类型 | 路径格式 |
|------|----------|
| 背景图 | `/uploads/images/scenes/backgrounds/{uuid}.webp` |
| 角色图 | `/uploads/images/scenes/characters/{uuid}.png` |

可通过管理后台上传图片获取具体路径。
