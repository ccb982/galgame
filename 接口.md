React 页面文件结构详解
## 文件之间的关系
1. 路由配置 ： App.jsx 中的路由配置将 URL 路径映射到页面组件
2. 页面渲染 ：页面组件 ( PageName.jsx ) 被路由渲染，成为用户直接看到的内容
3. 数据获取 ：页面组件通过 API 调用文件从后端获取数据
4. 状态管理 ：页面组件使用状态管理文件存储和管理数据
5. 样式应用 ：页面组件应用样式文件或 TailwindCSS 类来美化界面
## 实际项目中的例子
以 Galgame 播放器项目为例：

### 首页 (/)
- 页面主组件 ： src/pages/Home.jsx
- API 调用 ： src/api/game.js (getGames 方法)
- 状态管理 ：使用组件内部状态或 src/stores/useGameStore.js
- 路由配置 ： src/App.jsx 中 <Route path="/" element={<Home />} />
### 游戏详情页 (/game/:id)
- 页面主组件 ： src/pages/GameDetail.jsx
- API 调用 ： src/api/game.js (getGameById 方法) 和 src/api/save.js (getSaves 方法)
- 状态管理 ：使用组件内部状态或 src/stores/useSaveStore.js
- 路由配置 ： src/App.jsx 中 <Route path="/game/:id" element={<GameDetail />} />
### 游戏播放器 (/play/:gameId)
- 页面主组件 ： src/pages/Player.jsx
- API 调用 ： src/api/scene.js (getScene 方法) 和 src/api/save.js (createSave 方法)
- 状态管理 ： src/stores/useGameStore.js
- 路由配置 ： src/App.jsx 中 <Route path="/play/:gameId" element={<Player />} />
## 简化理解
如果把 React 页面比作一个房子：

- 页面主组件 ：房子的主体结构
- API 调用 ：连接房子和外部世界的管道
- 状态管理 ：房子的控制系统（如灯光、温度）
- 工具函数 ：房子里的工具和设备
- 样式文件 ：房子的装修和外观
- 路由配置 ：房子的地址和门牌号

## 后端 API 接口介绍

### 一、后端项目架构

当前后端项目是基于 **Spring Boot 2.7.18** 的 RESTful API 服务，主要负责 Galgame 游戏数据的存储、管理和提供。项目结构采用标准的 MVC 架构。

### 二、已实现的 API 接口

#### 1. 游戏管理接口（GameController）

**基础路径**：`/api/games`

| 请求方法 | 路径 | 功能描述 | 请求体 (JSON) | 成功响应 (200 OK) |
|---------|------|----------|---------------|-------------------|
| **GET** | `/` | 获取游戏列表（分页） | N/A | `{"content": [...], "totalElements": 10, "totalPages": 1}` |
| **GET** | `/{id}` | 根据 ID 获取游戏详情 | N/A | `{"id": 1, "title": "游戏标题", "description": "..."}` |
| **POST** | `/` | 创建新游戏 | `{"title": "游戏标题", "description": "...", "coverUrl": "...", "tags": "标签1,标签2"}` | `{"id": 1, "title": "游戏标题", ...}` |
| **PUT** | `/{id}` | 更新游戏信息 | `{"title": "新标题", "description": "..."}` | `{"id": 1, "title": "新标题", ...}` |
| **DELETE** | `/{id}` | 删除游戏 | N/A | `{"message": "删除成功"}` |

#### 2. 场景管理接口（SceneController）

**基础路径**：`/api/games/{gameId}/scenes`

| 请求方法 | 路径 | 功能描述 | 请求体 (JSON) | 成功响应 (200 OK) |
|---------|------|----------|---------------|-------------------|
| **GET** | `/` | 获取指定游戏的所有场景 | N/A | `[{"id": 1, "gameId": 1, "sceneKey": "start", "content": "..."}]` |
| **GET** | `/{sceneKey}` | 根据场景键获取场景详情 | N/A | `{"id": 1, "gameId": 1, "sceneKey": "start", "content": "..."}` |
| **POST** | `/` | 创建新场景 | `{"sceneKey": "start", "content": "{\"text\": \"...\", \"options\": [...]}"` | `{"id": 1, "gameId": 1, "sceneKey": "start", ...}` |
| **PUT** | `/{sceneKey}` | 更新场景信息 | `{"content": "{\"text\": \"新内容\", ...}"}` | `{"id": 1, "gameId": 1, "sceneKey": "start", ...}` |
| **DELETE** | `/{sceneKey}` | 删除场景 | N/A | `{"message": "删除成功"}` |

#### 3. 存档管理接口（SaveController）

**基础路径**：`/api/saves`

| 请求方法 | 路径 | 功能描述 | 请求体 (JSON) | 成功响应 (200 OK) |
|---------|------|----------|---------------|-------------------|
| **GET** | `/` | 获取存档列表（支持按游戏ID筛选） | N/A | `[{"id": 1, "gameId": 1, "sceneKey": "start", "visitorId": "..."}]` |
| **GET** | `/{id}` | 根据 ID 获取存档详情 | N/A | `{"id": 1, "gameId": 1, "sceneKey": "start", "variables": {...}}` |
| **POST** | `/` | 创建新存档 | `{"gameId": 1, "sceneKey": "start", "variables": {...}}` | `{"id": 1, "gameId": 1, "sceneKey": "start", ...}` |
| **PUT** | `/{id}` | 更新存档信息 | `{"sceneKey": "scene2", "variables": {...}}` | `{"id": 1, "gameId": 1, "sceneKey": "scene2", ...}` |
| **DELETE** | `/{id}` | 删除存档 | N/A | `{"message": "删除成功"}` |

### 三、后端服务状态

- ✅ **服务已启动**：运行在 `http://localhost:8080`
- ✅ **数据库连接**：已配置 MySQL 连接（`jdbc:mysql://localhost:3306/galgame_db`）
- ✅ **API 接口可用**：所有接口已实现并可正常访问

### 四、前端与后端的交互流程

1. **API 调用**：前端通过 Axios 实例发送请求
2. **游客 ID 添加**：请求拦截器自动添加 `X-Visitor-Id` 头
3. **后端处理**：后端接收请求，处理业务逻辑，操作数据库
4. **响应返回**：后端返回 JSON 格式的响应数据
5. **前端处理**：前端接收响应，更新 UI 或处理错误
